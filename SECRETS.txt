Before making the repository public, ensure you do *not* commit or expose any
sensitive information. To keep important details secret, do the following:

• Add a `.env` file to `.gitignore` so it doesn’t get checked into Git.
  - The file typically contains `DATABASE_URL`, API keys, or other credentials.

• Never commit real connection strings, passwords, or API tokens.
  - Rotate keys if they have accidentally been committed in past commits.

• Check your Git history for any secrets and remove them using tools like
  `git filter-branch`, `git rebase -i`, or the BFG Repo Cleaner.

• If you store credentials elsewhere (e.g. CI configs, deployment scripts),
  ensure those repositories remain private or scrub the values.

• When deploying, set environment variables on the host instead of hard-coding
  them in source.

• Treat any third‑party service credentials (e.g. MongoDB Atlas URI) as
  secrets and rotate them periodically.

Following these steps will let you keep the public repo clean and avoid
accidentally leaking anything sensitive.
